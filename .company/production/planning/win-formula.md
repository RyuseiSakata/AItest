# 勝ちの方程式 — 参考チャンネル実測データからの逆算(2026-07-05)

## 実測データ(各チャンネルのショート一覧 約40〜48本をサンプル)

### かわいいのすべて(AI動物、開設半年で総再生6,320万)
- 分布: 中央値 約5万回。100万回超えは40本中5本(約12%)
- 上位: 世界一かわいい落とし物 **2,277万回** / 世界一かわいい変身 870万回 / 彼女の甘えに彼氏の反応 347万回 / 猫たちが集まり始めた 191万回 / 猫生最大危機 133万回
- **上位2本だけでサンプル総再生の約7割**。完全なべき乗分布
- 勝ち型: 「世界一かわいい◯◯」シリーズ(日常の物や場面 → かわいい動物に化けるリビール構成)。ヒット後に同型を反復している
- タイトルは全て「!?」「www」の疑問・期待喚起型。固有名詞なし・説明なし

### 幸せワールド(AI寸劇、48本で総再生5,030万)
- 分布: 中央値 約4.5万回。100万回超えは48本中10本(約21%)
- 上位: 誕生日が1日違うとこうなるw **1,175万回** / 男友達の手を握ってみた結果 961万回 / 毎日妹をハグしてみた結果w 644万回 / 気になる子と柔道した結果w 513万回 / ママには内緒の動画w 387万回
- 勝ち型: 「**◯◯してみた結果w**」フォーマット+人間関係の緊張(友達/兄妹/彼氏彼女)。好奇心ギャップをタイトルで作り、本編で回収
- 同一「構成」を使い回しつつ、シチュエーション(脚本)は毎回変えている → 量産判定を回避しながら型の再現性を確保

## 導かれる勝ちの方程式
**再生数 = 打席数 × 型のヒット率 × 型の当たり値**、かつヒットはべき乗分布。
1. **型(フォーマット)が再現性の単位**。動画単発ではなく「シリーズ型」を設計し、型ごとに成績を測る
2. **打席数を確保**: 週6〜8本=月30打席。ヒット率12〜21%なら月3〜6本の100万回級が統計的に期待できる
3. **勝ち馬に全乗り**: 型が当たったら即、同型のバリエーションを連打(両チャンネルとも実践)
4. **タイトル/フック工学**: 冒頭1秒のビジュアルフックとタイトルの好奇心ギャップに全投資。説明的タイトルは全滅している
5. **70/20/10ポートフォリオ**: 実績型70% / 実績型の変形20% / 新型実験10%。実験が新しい勝ち型を発掘する
6. **量産判定対策**: 型は同じでも脚本・シチュエーション・オチは毎回変える(幸せワールド方式)

## 初期投入する「型」候補(たたき台)
- **型A「世界一かわいい◯◯」型(リビール型)**: 日常の場面→予想外のかわいい動物が出現。実績: 2,277万回の直接前例
- **型B「猫に◯◯してみた結果w」型(実験型)**: 幸せワールドの勝ち型を動物に移植。好奇心ギャップ+オチ
- **型C「動物寸劇・連載キャラ型」**: 固定キャラ(例: 世渡り上手な猫)の連続ドラマ。当たれば登録者が積み上がりIP化の出口。ホネ次郎型
- 運用案: A/B/Cを10本ずつ→2週間で成績比較→勝ち型に70%集中

## 週次運用ループ(再現性の担保)
1. 月曜: 前週の型別成績レビュー(再生数・視聴維持率・登録転換)
2. 配分決定: 70/20/10を成績で更新
3. 台本量産(秘書がAI生成)→ オーナーがトーン最終チェック(オリジナル視点の注入=BAN対策も兼ねる)
4. 生成・編集・投稿(将来的に制作部へ)
5. 48〜72時間後の初速データを型の成績表に記録

## 追記(2026-07-05): 生存者バイアス補正 — AI猫チャンネル35件の実測サンプリング
YouTube検索「AI猫」「AI cat videos」等でヒットしたチャンネルの登録者分布(検索露出バイアスあり=実際の失敗率はこれより高い):
- 日本語圏(n=23): 1万人以上 5件(22%) / 1,000〜1万人 12件(52%) / 1,000人未満 6件(26%)
  - 上位: AI猫にゃんこちん9.0万、四季ねこ(血液型あるある)6.9万、アジアねこ散歩4.4万
- 英語圏(n=12): 10万人以上 2件 / 1,000〜1万人 3件 / **1,000人未満 7件(58%)**
- 所見:
  1. 検索に出てくる(=比較的活発な)チャンネルですら、過半数が収益化ライン(登録1,000人)付近〜数千人で滞留
  2. 英語圏のほうが競争が苛烈(6割が1,000人未満)。日本語市場のほうが分布がマイルド
  3. **同一フォーマットでも明暗**: 「血液型あるある」型は四季ねこ6.9万 vs AI猫のいる生活2,160人。型だけでは勝てず、実行品質の分散が大きい証拠
  4. 登録1,000人未満の中には開設直後のチャンネルも含まれる(停滞と若さを区別できていない) → 深掘りするなら開設日・投稿数の突合が必要
